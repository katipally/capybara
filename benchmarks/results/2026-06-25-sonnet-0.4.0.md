# Sonnet 4.6, capybaraa 0.4.0, baseline vs capybaraa

Date: 2026-06-25. Model: `claude-sonnet-4-6`. Arms: `baseline` (bare agent, no plugin) vs
`capybaraa` (this repo, on). 3 runs per cell. Harness: `benchmarks/agentic/run.py`, judges:
`judge.py`. Activation verified per cell (capybaraa cells read back `on`, baseline empty).

## Headline

1. **Asks before coding.** On the underspecified CLARIFY tasks the bare agent guessed the
   whole spec and built it (on "add user settings persistence" it invented theme, font size,
   six languages, and notification toggles, ~238 lines median, none of it asked for);
   capybaraa wrote 0 lines and asked first. CLARIFY judge (0-3, separate model, no plugin):
   **capybaraa 1.5, baseline 0.0**.
2. **Leaner, same completeness.** On the five reducible tasks capybaraa was 6-50% smaller and
   the completeness judge scored **every cell 3/3 for both arms** (leaner, not less).
3. **No regression.** All nine deterministic gates passed **100% for both arms**: with a
   strong model the bare agent already nails the surgical tasks, and capybaraa holds that.

## Numbers

CLARIFY (0-3, higher = asked more before coding), n=6 per arm:

| arm | median | premature code written |
|-----|--------|------------------------|
| baseline | 0.0 | yes (settings: 238 LOC / 2 files, export: 17 LOC / 2 files) |
| capybaraa | 1.5 | none (0 files) |

Median LOC on the reducible tasks (lower is leaner), completeness 3/3 for both arms:

| task | baseline | capybaraa | delta |
|------|----------|-----------|-------|
| feat-rating (star widget) | 38 | 23 | -39% |
| feat-export (CSV) | 20 | 15 | -25% |
| feat-palette (Cmd-K) | 89 | 78 | -12% |
| hygiene-replace (parser) | 17 | 16 | -6% |
| complete-fixtest (bugfix) | 4 | 2 | -50% |

Deterministic gates, all 100% for both arms (n=3): correct, safe, reuse (lean-reuse),
native (lean-native), optimal (optimal-members), economy (economy-nofiller), sync
(sync-rename), hygiene (hygiene-replace), ran_check (complete-fixtest). The irreducible
tasks (lean-native, lean-reuse, optimal, economy, sync) tie on LOC too: nothing to cut.

Cost/time: capybaraa is a little pricier on the build tasks (the asking + done-gate
overhead, e.g. feat-rating $0.113 vs $0.097). On the ambiguous CLARIFY task it was
**cheaper and faster** ($0.21 / 21s vs $0.27 / 56s): asking beats building the wrong thing.

## Method notes / honesty

- Two fixes landed before these numbers were trusted, after a first pass surfaced
  artifacts: (a) the CLARIFY tasks now seed a tiny real app, so neither arm wastes a
  question on "where is the code?"; (b) CLARIFY tasks no longer get the "only the code you
  leave behind is measured" directive, which had been punishing the act of asking. With
  both fixed the CLARIFY result flipped to the table above.
- A handful of feature cells in the first pass hit a session rate limit and were re-run.
- Directional, not a leaderboard: a few tasks, one model. The CLARIFY tier is an LLM judge
  (auditable: fixed model, published rubric, every verdict lists asked vs missed).

Reproduce: `python3 run.py --selftest` then
`python3 run.py --all --arms baseline,capybaraa --models sonnet --runs 3`, then the judges,
then `python3 chart.py runs/<stamp> ../../assets/benchmark.svg`.
